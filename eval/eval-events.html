Template selection

Methodology

  • Chose template clicks representing high-quality clicks
  • All are saved in the identidrift::click_templates data object Section 4.2
  • Chose 4 total, 2 for Dall’s porpoise and 1 for each other species. Two were needed for Dall’s porpoise due to apparent increased variability in events.

Results

I can add some general remarks here about broad differences in the shapes of the different profiles and what features led us to select them as high-quality clicks.

Discussion points

This is important for those who are interested in seeing the exact profiles of these types of signals and could be helpful for people studying NBHF clicks in other regions, to compare to what we consider to be a high-quality click.

Template matching

Methodology

  • Used PAMguard MTC module to generate match scores for all database clicks.
  • The best match score along with the associated template ID is saved for each click in the identidrift::template data object.

Results

From Figure 3 we can look comparatively at how each species’ clicks matched against the four available templates.

The distributions show that:

  • Kogia seems to match best with it’s template, since the distribution mean is well above the other two. This indicates also that the other species don’t match well with it.
  • For the harbor porpoise template, the flattened distribution of harbor porpoise appears to have a higher mean than the other species. Another indication that this template is working well, like the Kogia template
  • Mixed results for Dall’s porpoise. Not seeing the same trends that would indicate that it is being matched well with it’s templates.

Discussion points

  • Harbor porpoise and Kogia templates is doing a good job of matching with their respective species while excluding others.
  • Will the poorer matching of Dall’s porpoise be reflected in the classifier’s results as well? This could say something about this species, such as that it exhibits greater variability and has a signature that is harder to constrain.

Define events

Methodology

  • Used custom script to label the events in each database. Required a match score threshold to be given for each template (0.45-0.6)
  • Also required setting maximum separation between clicks in the same event, given as 120 sec, and minimum number of clicks per event, given as 3.
  • Script then labeled events in the database
    • Only clicks that were seen as part of an event were included in the training data set.
    • Clicks not included in events were not used for any further analysis.

Results

Recall and precision would be common metrics for assessing the performance of our event definition procedure. Recall (Table 1) would be estimated by counting clicks that possessed high match scores but which did not make it into the training set (false negatives). This would occur when high-quality clicks were isolated and did not occur temporally with enough other clicks to warrant declaring an event. This means they will have been ommitted from the training set. This does appear to be a rather small component since as we see in Table 1 recall is around ~90% across all species.

Precision would be more difficult to estimate. There is a region of overlap in the score distributions of the data that was and was not included (see Figure 4). This suggests sort of a “grey” area where the match is sometimes poor, but with enough other matches could nonetheless warrant an event?

To investigate further I plot to see if there is a relationship between the mean match score of an event and the size of an event. Figure 5 shows that larger events tend to have higher mean match scores. Looks like a pretty strong positive relationship.

Discussion points

  • Recall is high, which we expected because we deliberately lowered the threshold out of concern for missing valuable data in our training set.
  • Precision is harder to estimate. We might expect precision to have been diminished since we tailored our model to maximize recall.
